# Token 经济学（AI API 成本优化）

## 定义

在 AI 应用开发中，**Token 消耗直接等于成本**。Token 经济学就是研究如何在不降低输出质量的前提下最小化 Token 消耗。

## 来源视角
- **9 行 CLAUDE.md 让 token 直降 63%**（220 号）：精简系统提示词
- **50 元 Codex vs 100 刀 Claude Opus**（129/149 号）：模型选型
- **Coding plan token 包复用方案**（158 号）：避免重复消耗
- **Codex Proxy 多账号管理**（133 号）：绕过限制 + 负载均衡

## 优化层级（按 ROI 排序）
| 层级 | 方法 | 节省幅度 | 难度 |
|------|------|---------|------|
| L1 提示词精简 | CLAUDE.md 瘦身 | 40-63% | 低 |
| L2 模型选型 | 用 Codex 替代 Claude 执行 | 60-80% | 中 |
| L3 缓存复用 | 相似查询命中缓存 | 30-50% | 中 |
| L4 架构设计 | SubAgent 模式 | 50-70% | 高 |
| L5 多账号 | Token Pool / Proxy | 无限扩展 | 高 |

## 实践建议
1. 先做 L1（精简 CLAUDE.md），5 分钟见效
2. 再做 L2（Codex SubAgent），性价比最高
3. 监控每次对话的 Token 消耗，建立基线
4. 设定预算上限，超了自动降级到便宜模型
