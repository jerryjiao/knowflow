# KnowFlow 开源指南：3 分钟上手 + 10 个踩坑记录

## 写在前面

KnowFlow 开源之后，收到最多的问题不是"这东西能干嘛"，而是"装不上怎么办"。

这篇文章就是为解决这个问题写的。前半部分是安装和首次运行的完整流程，后半部分是我自己和早期用户实际遇到的 10 个问题，附排查思路和解决方案。

如果你第一次接触 KnowFlow，建议先通读安装部分再动手；如果已经装过但卡住了，直接跳到踩坑区对号入座。

---

## 3 分钟安装指南

### 前置要求

KnowFlow 的运行依赖以下环境，请提前确认：

| 依赖 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | >= 18（推荐 LTS） | CLI 和核心流程 |
| Python | >= 3.10 | 知识图谱构建（graph_builder.py） |
| 智谱 AI API Key | — | 文本抽取和向量化 |

API Key 在 [open.bigmodel.cn](https://open.bigmodel.cn) 免费申请，注册后即可拿到。

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/jerryjiao/knowflow.git
cd knowflow

# 2. 安装依赖
npm install

# 3. 初始化配置
npx knowflow init
# 这会生成 .knowflowrc 配置文件
# 也可以手动：cp .knowflowrc.example .knowflowrc

# 4. 编辑配置，填入 API Key
nano .knowflowrc  # 或用你喜欢的编辑器

# 5. 验证安装
npx knowflow --version    # 输出 0.1.0
npx knowflow health       # 检查各项状态
```

`health` 命令会检查 Wiki 的健康状态：断链检测、空文件扫描、孤立页面发现。四项全绿说明知识库状态良好。

配置文件 `.knowflowrc` 中最关键的两个字段：

```yaml
ai:
  provider: "zhipu"          # 支持 zhipu / openai / anthropic
  api_key: "your-key-here"   # 替换成你的实际 Key
```

### 第一次 Ingest

环境就绪后，试着导入一个数据源：

```bash
npx knowflow ingest https://karpathy.github.io/llm-wiki/
```

正常运行会看到完整的 Fetch → Extract → Compile → Graph → Vector 流程走完，日志里会输出创建的页面数量。到这里，KnowFlow 就算跑通了。

接下来可以用这些命令探索你的知识库：

| 命令 | 功能 |
|------|------|
| `npx knowflow query <文本>` | 向量检索，语义搜索 Wiki 内容 |
| `npx knowflow graph` | 构建并打开知识图谱 |
| `npx knowflow status` | 查看 Wiki 统计信息 |
| `npx knowflow health` | Wiki 健康检查（断链/空文件/孤立页面） |

## 10 个踩坑记录

下面是实际使用中遇到的高频问题。按出现频率从高到低排列。

### 坑 1：Node 版本太低

**表现**：`SyntaxError: Unexpected token '??'` 或 `Cannot use import statement outside a module`

**原因**：Node.js 版本低于 18，不支持 ES Module 和可选链操作符（`?.`）。这是最高频的安装失败原因。

**解决**：

```bash
# 用 nvm 切换（推荐）
nvm install --lts
nvm use --lts

# 或直接安装
brew install node@20
```

切换后记得重新 `npm install`。

### 坑 2：API Key 格式错误

**表现**：请求返回 401 Unauthorized

**原因**：复制 Key 时带入了首尾空格，或者误用了 Secret Key 而非 API Key。

**解决**：回到智谱控制台重新复制，粘贴后检查 `.knowflowrc` 中 `api_key` 字段的值是否干净。一个常见陷阱是从终端复制时多选中了一个换行符。

### 坑 3：Python 版本不足导致图谱空白

**表现**：`npx knowflow graph` 生成的页面一片空白

**原因**：`graph_builder.py` 需要 Python 3.10+，系统自带的 Python 版本可能不够。

**解决**：

```bash
python3 --version  # 先确认当前版本
# 如果低于 3.10，升级 Python
brew install python@3.12
```

### 坑 4：中文文件名编码问题

**表现**：Wiki 中的中文链接返回 404，但没有任何报错——静默失败。

**原因**：终端编码不是 UTF-8，文件名在写入时发生乱码。

**解决**：在 `.bashrc` 或 `.zshrc` 中添加：

```bash
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
```

改完后 `source` 一下或重启终端。

### 坑 5：Jina Reader 限流

**表现**：ingest 过程中报 429 Too Many Requests

**原因**：KnowFlow 默认使用 Jina Reader 抓取网页内容，免费版有约 200 次/小时的请求限制。批量导入时容易触发。

**解决**：间隔一段时间后重试，或减少单次 ingest 的页面数量。大规模导入建议分批进行。

### 坑 6：Shell 脚本无执行权限

**表现**：运行 `scripts/` 下的脚本时报 Permission denied

**原因**：`git clone` 下来的文件默认不带执行权限。

**解决**：

```bash
chmod +x scripts/*.sh
# 或者每次用 bash 显式调用
bash scripts/pipeline.sh
```

### 坑 7：Ingest 中途中断导致索引损坏

**表现**：`query` 返回空结果，但 Wiki 里明明有内容。

**原因**：上次 ingest 跑到一半被 Ctrl+C 中断了，向量索引处于半写入状态，留下了脏数据。

**解决**：删除索引缓存文件后重新 ingest：

```bash
rm wiki/.vector-index.json wiki/.embed-cache.json
npx knowflow ingest <your-url>
```

这两个文件是运行时自动生成的，删除后会触发完整重建。

### 坑 8：子进程超时退出

**表现**：pipeline 运行中突然退出，exit code 134（SIGABRT）

**原因**：某个子进程（通常是网络请求或 Python 调用）超时，触发了 SIGABRT 信号。代码层已经加了容错处理（timeout + WARN 日志），但在网络不稳定的环境下仍可能出现。

**解决**：检查网络连接，确认 API 服务可访问。如果是偶发问题，直接重试即可。

### 坑 9：macOS Gatekeeper 拦截

**表现**：首次运行弹出"无法验证开发者"的系统提示。

**原因**：macOS 的安全机制会阻止未签名的可执行文件运行。

**解决**：

```bash
# 移除隔离标记
xattr -d com.apple.quarantine $(which knowflow)
```

或者在系统设置 → 隐私与安全性 中点击"仍要打开"。

### 坑 10：npm 包名已被占用

**表现**：如果你想发布自己的定制版本，遇到 403 Package name already exists。

**原因**：npm 上的包名是全局唯一的，`knowflow` 名字已被占用。

**解决**：发布前先在 [npmjs.com](https://www.npmjs.com) 搜索目标名称确认。建议使用 scoped 包名，如 `@yourname/knowflow`。

## 几条实用建议

1. **先少量再批量**。别一上来丢 100 个 URL 进去，先用 3-5 个跑通全流程，确认环境没问题再扩大规模。

2. **手动编辑 Wiki 后记得重建索引**。直接修改 `wiki/` 下的文件不会自动更新向量索引，需要重新 ingest 或手动触发 rebuild，否则 query 结果可能不准确。

3. **定期备份 `wiki/` 目录**。这是你的核心知识资产。注意 `.gitignore` 默认排除了 `wiki/` 和 `raw/` 等数据目录——如果要用 git 备份，需要手动修改 `.gitignore` 或单独用其他方式备份。

4. **善用 `knowflow graph` 做知识发现**。图谱视图能帮你发现文档之间的意外关联，偶尔看看会有惊喜。

5. **自定义 extract prompt**。`.knowflowrc` 中可以修改抽取 prompt，适配你的特定领域（比如法律、医疗、金融），效果会好很多。
