# KnowFlow 开源了：3 分钟上手 + 10 个踩坑记录

## 前言

这篇文章专门讲"怎么装"和"哪里会炸"。教程写得再漂亮，读者上手时还是会遇到各种奇葩问题。Node 版本不对、API Key 配错、中文路径编码、Shell 权限……坑多得离谱。

所以这篇是安全网——虽然网眼可能有点大，漏几条鱼。别指望兜住所有问题。

## 3 分钟安装

### 前置要求

- Node.js >= 18（推荐 LTS）
- Python 3.10+（图谱构建用）
- 智谱 AI API Key（免费申请 open.bigmodel.cn）

### 安装

```bash
# 1. 克隆仓库
git clone https://github.com/jerryjiao/knowflow.git
cd knowflow

# 2. 安装依赖
npm install

# 3. 复制配置文件
cp .knowflowrc.example .knowflowrc
# 编辑 .knowflowrc，填入你的 API Key

# 4. 验证安装
npx knowflow --version    # 查看版本号
npx knowflow health       # 检查状态
```

`.knowflowrc` 最关键的就是 `ai.api_key`。填错了后面全白费。血泪史告诉你别问怎么知道的。

### 第一次 Ingest

```bash
npx knowflow ingest https://karpathy.github.io/llm-wiki/
```

正常会看到 Fetch→Extract→Compile→Graph→Vector 全流程跑完，4-5 个页面创建成功。看到这个——恭喜！

没看到？欢迎来踩坑区对号入座。

## 10 个踩坑记录

### 坑 1：Node 版本太低

**报错**: `SyntaxError: Unexpected token '??'` 或 `Cannot use import statement`

**原因**: Node 太老（<18），ES Module 和可选链跑不动。

**解决**: `brew install node@20` 或 `nvm install --lts`

### 坑 2：Python 缺失图谱空白

**现象**: `knowflow graph` 打开白茫茫一片

**原因**: `graph_builder.py` 要 Python 3.10+

**解决**: `python3 --version` 确认版本，不够就升级

### 坑 3：API Key 格式错误

**报错**: 401 Unauthorized

**原因**: 复制 Key 多了空格或把 Secret Key 当 API Key 用

**解决**: 去控制台重新复制，仔细检查首尾空格

### 坑 4：中文文件名编码

**现象**: Wiki 中文链接 404，不报错、静默失败

**原因**: 终端 UTF-8 不一致，文件名乱码

**解决**: shell 加 `export LANG=en_US.UTF-8` 和 `LC_ALL=en_US.UTF-8`

### 坑 5：Jina Reader 被限流

**报错**: 429 Too Many Requests

**原因**: 免费版限制约 200 次/小时，ingest 太猛

**解决**: 等一小时再试，或切换内置 fetch

### 坑 6：Shell 脚本无执行权限

**报错**: Permission denied

**原因**: clone 下脚本默认无执行权限

**解决**: `chmod +x scripts/*.sh` 或直接 `bash scripts/pipeline.sh`

### 坑 7：SIGABRT 崩溃

**现象**: pipeline 跑一半突然退出 exit code 134

**原因**: 子进程超时 SIGABRT 信号

**解决**: 代码已加容错（timeout + WARN），还出现就检查网络、增大超时

### 坑 8：向量索引损坏

**现象**: query 返回空结果但 Wiki 有内容

**原因**: 上次 ingest 中途被中断（Ctrl+C），向量库半写状态脏数据

**解决**: 删除 `.vector-index.json` 和 `.embed-cache.json`，重新 ingest 触发重建

### 坑 9：macOS Gatekeeper 拦截

**现象**: 首次运行弹出"无法验证开发者"

**原因**: Apple 安全机制阻止未签名可执行文件

**解决**: `xattr -d com.apple.quarantine /usr/local/bin/knowflow` 或系统设置 → 隐私 → "仍要打开"

### 坑 10：npm 包名冲突

**报错**: 403 Package name already exists（如果你尝试发布自己的包）

**原因**: 有人先占了名字

**解决**: 发布前先 npmjs.com 搜名字，建议 fork 改 scope 如 `@yourname/knowflow`。KnowFlow 目前可用，别抢 😂

## 最佳实践

1. **小批量先测** — 别一上来丢 100 URL，先 3-5 个确认跑通
2. **定期 rebuild index** — 手动编辑 Wiki 后重建索引，不然查询可能不准
3. **备份 wiki/** — 核心资产，定期 git commit
4. **善用 .gitignore** — raw/ 和向量缓存不提交仓库，膨胀很快
5. **自定义 prompt** — .knowflowrc 改 extract prompt 适配你的领域
6. **定期看图谱** — `knowflow graph` 找意外关联，偶尔有惊喜
