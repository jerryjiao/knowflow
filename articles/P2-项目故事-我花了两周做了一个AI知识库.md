# 我花了两周做了一个 AI 知识库，这是效果

## 先说结果

做了两周。Wiki 里躺着 **20+ 篇**文章，知识图谱随内容量动态增长——说"两周"其实不准确，断断续续加起来大概十天？剩下的时间在踩坑。好累。

不多。但它是活的。活的就好死代码没意义。没意义就是垃圾。垃圾该扔。死了就白写了。白写最伤心。伤心欲绝 凉了。

我希望可以做一个知识库，这样就可以方便学习收藏的东西。


## 起因：我的信息焦虑

我是一个前端开发，最近在考虑转型。转型的第一步不是学新框架——是**把脑子里和收藏夹里的碎片信息变成可用知识**。

我关注的东西很杂：AI 工具、开发者效率、独立开发、技术创业、前端框架……每天刷 Twitter 看公众号读技术博客看到好的就收藏。

然后？然后就没有然后了。

微信收藏几十篇文章上次打开是三月份。浏览器书签？几百个，90% 连标题都想不起来为什么存。这太荒谬了。

OpenText 报告说 80% 知识工作者经历信息过载。

问题不在没有好信息。在**信息进来了但从来没被消化过**。就像一直吃东西从不消化——到头来撑得难受营养不良还胖。

## 我试过的（都失败了）

**Notion 建数据库。** 建了三个 workspace——听起来挺有决心吧？每个填不到 10 条记录就放弃。每次手动分类打标签建关联太累了。人的惰性可怕：收集成本大于"以后有用"的预期收益就永远不会发生。

**Obsidian + 插件。** 配置两天装一堆插件漂亮界面爽三天。然后中文支持一塌糊涂双向链接中文内容经常断掉。本质还是编辑器——不帮你整理只给你更好看的本子。

**直接丢给 ChatGPT/Claude。** 上传文档问问题能用。但问题——下次问同样的问题它**重新来过**。没记忆没积累没上下文。像每天早上跟同事打招呼他每天都问"我们认识吗？"

这太扯了。受够了这种"伪效率"。工具一大堆效率没涨。焦虑倒是涨了不少。

## KnowFlow 的诞生

后来看到 Andrej Karpathy 的 LLM Wiki 想法。Karpathy——前 Tesla AI 总监 OpenAI 联合创始人——提出跟 RAG 完全不同的思路：**别每次查询去原始文档翻先把知识编译好放在那用的时候直接拿。**

RAG 是临时工。LLM Wiki 是全职图书管理员。这个类比打动了我。决定自己做。

### 技术选型（每一步都是取舍）

| 选型 | 选择 | 为什么 |
|------|------|--------|
| 运行时 | Node.js + Python 混合 | 熟悉 JS Python 数据处理强 |
| AI 引擎 | 智谱 AI (GLM-4-Flash) | 中文 embedding 效果好 免费够用 |
| 存储 | Markdown + JSON | 本地优先 不需要数据库服务 |
| 可视化 | vis-network (vis.js) | 知识图谱力导向图可视化（自包含 HTML，无需服务器）|
| 向量检索 | 智谱 embedding-3 | 中文语义搜索 |
| CLI 框架 | Commander.js | 轻量够用 |

智谱 AI 是最意外收获。一开始想直接上 OpenAI，跑完数据后发现中文效果一般般。换成智谱再跑一遍——**好多了**。实测结果。

## 它现在能做什么

六个命令：

`knowflow init` — 初始化配置文件模板（`.knowflowrc` + `.env`）。

`knowflow ingest URL` — 丢 URL 进去，自动识别来源（Twitter/小红书/微信/网页/YouTube），提取全文保存为原始素材。支持 `-s` 指定来源类型。

`knowflow graph` — 基于 Wiki 中的 `[[wikilink]]` 关系，用 **vis-network (vis.js)** 构建力导向知识图谱并在浏览器中打开。自包含 HTML，无需服务器。节点大小=引用次数，颜色=类型，可拖拽缩放。

`knowflow query "问题"` — 混合检索（**向量语义搜索 + 关键词匹配**），从编译好的 Wiki 中查找相关内容并返回来源。

`knowflow health` — Wiki 健康检查：断链检测、空文件扫描、孤立页面发现。

`knowflow status` — 显示 Wiki 统计概览（文章数、素材数、向量索引状态、API 配置）。

## 踩过的坑

**ingest 提取失败静默吞错。** URL 下载或内容提取失败时，早期版本不报错只存空文件。后续 ingest 拿到空文件就跑出垃圾 Wiki 页面。加了文件大小校验和提取方法回退（Jina Reader → 直接 curl → 保存原始链接）。

**Wiki 链接断裂。** AI 提取实体时偶尔生成空名称，导致 `[[entities/,]]` `[[concepts/,]]` 这种空链接。写了清理脚本批量修复。

**npm 包名冲突。** 一开始想叫别的名字，后来发现 knowflow 这个名字 GitHub 和 npm 都可用。**教训：名字第一时间占位，想好了就注册。**

**pipeline 各步骤依赖外部 Agent。** ingest 只负责「URL→原文」，后续的 AI 提取、Wiki 生成、图谱构建需要配合 LLM Agent 使用，不是纯 CLI 一条龙。这个设计选择让每一步都可以人工干预，但也增加了上手复杂度——所以后面写了 P5 上手指南。

- **P4**: 知识图谱 + 向量检索原理与实现
- **P5**: KnowFlow 开源了— 3 分钟上手 + 10 个踩坑
- **P6**: 做完之后对 AI 知识管理的思考

**边做边教。** AI 工具遍地开花的时代理解原理比会用工具更有价值。工具会变原理不会。

而且做完发现最有意思的不是最终那个工具——是**你看着零散信息一点点变成结构化知识的过程**。
