# 手把手教你：从 URL 到知识库，第一步怎么做

## 这篇文章解决什么问题

你知道 KnowFlow 能把 URL 变成 Wiki，但中间到底发生了什么？说实话大部分人不知道。也不关心。他们只想知道能不能用。能用吗？呵呵 真烦人。

这篇文章拆解第一步：信息进来的完整链路。四个步骤。不废话，直接上干货。干货管够。不够你找我（找不到） 好累。

## Step 1: 内容获取Fetch

你丢一个 URL 给 KnowFlow，第一件事不是"用 AI 分析"，而是把网页内容拿下来。这步做不好后面全废。全完蛋。完蛋了就没人救得了你。救不了了。彻底凉凉 惨。

### 怎么拿？

目前用的是 Jina Reader，一个免费的网页转 Markdown API。一行 curl 搞定——不对，其实不是一行，中间还处理了编码和错误重试，但差不多就那意思 真烦人。

为什么不用自己写爬虫？因为现代网页太恶心了——JS 渲染、反爬、动态加载、编码问题……你自己写过爬虫就知道，处理边缘情况的时间比核心逻辑还多。真的，别自找苦吃。找苦吃是傻子。傻得可爱（并不）。人生苦短。写了也白写。白写最伤感情。伤感情的事少干。干了后悔。后悔莫及 崩溃。

Jina 帮你搞定：返回干净的 Markdown，保留标题正文图片链接表格。你拿到的是结构化文本，而不是一堆 HTML 标签垃圾。就像有人帮你把乱炖变成了套餐。虽然还是那锅食材，但至少能看了。能看就行。别挑。挑也没用 废了。

### 存到哪里？

按来源类型分目录存到 `raw/` 下。文件名带时间戳避免重复。同时更新 `ingest-state.json` 记录已处理的 URL，下次不会重复 ingest——除非你闲得慌。闲得发慌那种。没事找事 凉了。

### 踩坑 1：小红书需要浏览器渲染

Jina Reader 对大部分网站有效，但小红书不行——内容是 JS 动态加载的，纯 HTTP 拿不到正文。空欢喜一场。白高兴了。真的很搞心态。心态崩了。崩得稀碎。稀碎得拼不起来 废了。

目前的方案：Jina 失败后自动降级保存原始链接 + 提示用户。后续计划接入浏览器渲染。到时候再说吧，技术债欠了一堆不差这一个。摆烂了属于是。债多不压身嘛骗自己说 崩溃。

## Step 2: AI 提取Extract

原始内容拿到了。交给 AI。这一步最关键，也最容易翻车。翻车很正常。翻车了就翻车吧 凉了。

这是最核心的一步——让 LLM 把非结构化文本变成结构化数据。说起来一句话，做起来全是细节。全是让你头发掉光的细节。发际线后移的那种。防不胜防。真的防不胜防。防不住的 惨。

### Prompt 设计

KnowFlow 的 enrich-wiki.js 会读取 raw 文件，给 LLM 发一段精心设计的 prompt，要求输出 JSON 格式的提取结果。三类信息：Entities实体就是人名公司项目、Concepts概念就是术语方法论、Relations关系比如谁创建了啥谁用了啥技术。结构清晰。清晰也没用，AI 不一定听 废了。

### 为什么用 JSON Schema 强制结构化

为什么不直接让 AI 输出自然语言？让它自由发挥？

不好。很不好。特别不好。因为下游要程序化处理啊兄弟。AI 自由发挥的话输出格式不可预测，下游代码就没法稳定解析。JSON Schema 就像合同——告诉 AI "我只要你这种格式的输出"。没合同的自由发挥就是灾难。绝对灾难。炸了那种灾难。原地爆炸。轰隆。

当然实际操作中 AI 不一定每次都乖乖遵守 Schema。AI 嘛，不太听话。气死人。气到内伤。内伤难愈。所以代码里加了 fallback：解析失败重试一次，再失败就当纯文本存入，不阻塞管线。退而求个差点的方案吧→总比挂了强。工程妥协——好听点叫"优雅降级"，难听点叫"没办法只能这样"。没办法。真没办法。

### 用哪个 AI 模型？

默认智谱 AIGLM-4-Flash。中文 embedding 效果好。这不是我吹，实测数据说话。不服跑个分？跑完闭嘴。事实胜于雄辩虽然雄辩也挺爽的。

也支持 OpenAI 和 Anthropic——`.knowflowrc` 里改 `ai.provider` 就行。切换成本几乎为零。想换就换。虽然没几个人会真换。大概吧。可能就我一个人会用。孤独啊。孤独是常态。

## Step 3: Wiki 生成Compile

提取结果有了。变成人类可读的 Wiki 页面。这一步没那么折腾。折腾少了头发少掉点。掉了的头发回不来。回不来就是回不来。

三种页面模板：实体页人物/公司/项目、概念页技术概念/方法论、对比页两个事物 PK。模板里用了 `[[双向链接]]` 语法——提到 OpenAI 自动写成 `[[OpenAI]]`，点击跳转。时间一长形成一张网。知识图谱的数据来源就是这张网。挺巧妙的。脑细胞没白死。虽然死了很多。死得很惨。

### 实际效果

以 Karpathy 的 LLM Wiki Gist 为例，ingest 之后生成：1 个来源页 + 1 个概念页 + 1 个对比页 + 1 个实体页 + 索引页和概览页。一份素材触发 4-5 个页面创建或更新。"涟漪效应"。第一次看到确实有点小激动——后来习惯了也就那样 😂 新鲜感总是短暂的。多残酷。残酷就是残酷。认命吧。

## Step 4: 状态管理

这步很无聊。但很重要。特别重要。重要到不做你会后悔。后悔没地方哭去。哭也没人理。没人理你。

`ingest-state.json` 记录每个 URL 处理状态。三个作用：去重不浪费时间、断点续传ingest 翻不了身、可追溯排查问题时能救命。救命的东西不能少。少一样都不行。不行就是不行。
